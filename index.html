      Ciclo de una petición

      Cada petición deja el origen en paz

      El objetivo es dar tiempos frescos sin castigar a la API interna de TUSSAM. Cache con TTL, una sola llamada en vuelo por parada y reintentos que respetan el Retry-After del origen.

      1. 01
        Valida y limita

        Comprueba coordenadas y formato de parámetros, y aplica el rate limiting combinado por IP y por dispositivo antes de tocar nada.

      2. 02
        Busca la parada

        Prefiltra por bounding box y afina con Haversine sobre las 967 paradas cargadas en SQLite, sin salir a la red.

      3. 03
        Sirve desde cache

        Si hay tiempos de menos de 60 segundos, responde al instante. El origen solo se consulta cuando la cache expira.

      4. 04
        Consulta con single-flight

        Cien clientes preguntando por la misma parada generan una única petición saliente; el resto espera su resultado.

      5. 05
        Resiste y responde

        Reintenta ante 429/5xx, cae a la última respuesta buena si el origen falla, y devuelve JSON o GeoJSON señalizando el estado.

      Fiabilidad

      El origen falla.
      La API responde.

      reddelineas.tussam.es devuelve 429 con facilidad y a veces cambia el formato de la respuesta. Cada defensa está pensada para que el cliente final nunca herede esos problemas.

      El origen hace La API responde
      Rate limit agresivo

      Bloquea peticiones frecuentes con HTTP 429.

      Cache con TTL de 60 s

      Los tiempos se sirven desde SQLite; el origen solo se consulta al expirar.

      Se satura con ráfagas

      Muchas llamadas iguales a la vez lo tumban.

      Single-flight por parada

      Las peticiones concurrentes a una parada comparten una sola llamada saliente.

      Caídas intermitentes

      Responde 5xx o deja de contestar sin avisar.

      Cache stale marcada

      Sirve la última respuesta válida con stale: true y cached_at.

      Formato propietario

      Coordenadas en E6 y contrato sin documentar.

      Normalización estable

      Grados decimales, GeoJSON opcional y tiempos_status por parada.

      Superficie pública

      Endpoints que resuelven casos reales

      Desde «¿qué bus me viene?» hasta pintar la red entera en un mapa. Todo en JSON limpio, con coordenadas en grados decimales y direcciones postales completas.

      GET Recomendado

      /cercanas

      El endpoint agregado: paradas cercanas con sus tiempos de llegada en una sola llamada. Filtros por línea, sentido, orientación y tiempo máximo. Salida JSON o GeoJSON.

      • Una petición, todas las paradas cercanas y sus buses
      • Estado del origen por parada (tiempos_status)
      • GeoJSON nativo para Leaflet o MapLibre
      • GET /paradas Las 967 paradas con coordenadas, calle, número y código postal. público
      • GET /paradas/{codigo}/tiempos Buses en camino a una parada: línea, destino, minutos y vehículo. público
      • GET /lineas Líneas con su color oficial, horarios y recorrido por sentido. público
      • GET /health Estado de la API y la base de datos para Docker y balanceadores. público
      • POST /sync/* Sincronización de paradas, líneas y geocodificación bajo API key. api key
